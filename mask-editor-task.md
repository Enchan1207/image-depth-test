# マスクエディタの実装

## 概要

各レイヤのマスクについて、ユーザによる編集機能を設けたい。

## UI

### 編集コントロールの追加

現在画面右下ペインに表示されているレイヤの各項目に編集ボタンを追加する。
編集ボタンをクリックするとレイヤエディタが起動する。

### レイヤエディタ

レイヤエディタは大きく2つのペインから構成される、既存のメインウィンドウとは独立したウィンドウである。
左ペインは入力画像に対するマスクパターンの編集UI、右ペインはその結果として得られる画像のプレビューUIである。

- 左ペイン
  - マスクパターンがモノクロ画像で表示される。
  - PencilKitを活用することもできるが、最小限は「鉛筆・消しゴムの切り替え」「ツールの太さ選択」ができれば良い。
  - 現時点ではマスクパターンは2値とするが、内部的にはboolではなく数値として持って置けると良い。
- 右ペイン
  - 左ペインで作成したマスクに従って画像を切り抜き、その結果を表示する。
  - original画像とmasked画像を半透明にして重ねて表示する。

各ペインはscrollableである(既存のプレビュー画面と同様)。

エディタを閉じる際、保存するかどうかをダイアログでユーザに確認する。

## 挙動

深度スライダを動かす際、すでに編集されているものがある場合は警告を表示し、ユーザの承認が得られれば全マスクを再構成する。

## Tasks

- [ ] 既存のレイヤ一覧UIを確認し、各レイヤ行にマスク編集ボタンを追加する。
  - 編集ボタンは対象レイヤ、入力画像、深度レンジ由来の初期マスクを渡して独立したマスクエディタウィンドウを開く。
  - 画像未選択、深度推定未完了、レイヤレンダリング未完了など、編集できない状態ではボタンを無効化する。
- [ ] レイヤごとの編集済みマスクを保持できるデータ構造を追加する。
  - 2値表示を前提にしつつ、内部値は将来の濃淡マスクに備えて数値配列またはグレースケール画像として扱う。
  - 既存の深度レンジ由来マスクとユーザ編集済みマスクを区別できる状態を持たせる。
- [ ] メインウィンドウから独立したマスクエディタウィンドウの起動・管理を実装する。
  - SwiftUIの別ウィンドウとして開く方法を決める。
  - 同一レイヤのエディタを複数開けるか、既存ウィンドウを前面に出すかを決める。
  - ウィンドウに対象レイヤ名、入力画像、初期マスク、保存コールバックを渡す。
- [ ] マスクエディタウィンドウの基本レイアウトを追加する。
  - 左ペインにマスクパターン編集キャンバスを表示する。
  - 右ペインにマスク適用後のプレビューを表示する。
  - ウィンドウ内にツールバーまたは上部コントロールを置き、保存・破棄・ツール切り替えを操作できるようにする。
  - 各ペインをスクロール・ズーム可能にする。
- [ ] PencilKitを使ったマスク編集キャンバスを実装する。
  - 入力画像サイズに対応したモノクロのベースマスク画像を表示する。
  - ベースマスク画像の上にPencilKitの描画レイヤを重ねる。
  - 鉛筆と消しゴムを切り替えられるようにする。
  - ツールの太さを選択できるようにする。
  - 編集中の描画変更を検知し、プレビュー更新へ通知する。
- [ ] PencilKit描画をマスク画像へ反映する処理を実装する。
  - `PKDrawing`を入力画像と同じピクセル寸法の画像へラスタライズする。
  - ベースマスクと描画差分を合成し、8bitグレースケールの`CGImage`マスクを生成する。
  - 現時点では2値化しつつ、内部表現は0...255の濃淡値を扱える形にする。
  - 消しゴムをPencilKitのストローク削除として扱うか、黒ペンによるマスク削除として扱うかを実装方針として固定する。
- [ ] マスク適用プレビューを実装する。
  - 編集中マスクに従って元画像を切り抜いた画像を生成する。
  - original画像とmasked画像を半透明で重ねて表示する。
  - 編集操作に追従してプレビューを更新する。
- [ ] マスクエディタウィンドウの保存・破棄フローを実装する。
  - エディタを閉じる際、未保存変更がある場合は保存するかどうかを確認する。
  - 保存時は対象レイヤの編集済みマスクとして反映する。
  - 破棄時は編集前のマスク状態に戻す。
  - 保存または破棄後にエディタウィンドウを閉じる。
- [ ] 編集済みマスクを既存のレイヤレンダリング処理に接続する。
  - 編集済みマスクがあるレイヤでは、深度レンジ由来マスクではなく編集済みマスクを使う。
  - 編集済みマスクがないレイヤでは、従来どおり深度レンジ由来マスクを使う。
  - 編集済みマスク保存後、メインウィンドウのcutoutとプレビューを再生成する。
- [ ] 深度スライダ変更時の警告を追加する。
  - 編集済みマスクが存在する状態で深度レンジを変更しようとした場合、全マスクを再構成する旨を確認する。
  - 承認された場合は編集済みマスクを破棄し、深度レンジ由来マスクを再生成する。
  - キャンセルされた場合は深度レンジ変更を取り消す。
  - 開いているマスクエディタウィンドウがある場合、保存・破棄・キャンセルの扱いを決める。
- [ ] 画像サイズ変更・入力画像差し替え時の扱いを決めて実装する。
  - 既存の編集済みマスクを破棄するか、リサイズして引き継ぐかを決める。
  - 必要に応じてユーザ確認を表示する。
  - 開いているマスクエディタウィンドウがある場合は閉じるか、対象画像の変更を通知して編集を中断する。
- [ ] 実装後にビルドし、主要操作を確認する。
  - メインウィンドウから独立したマスクエディタウィンドウを開けること。
  - 鉛筆・消しゴム・太さ変更が反映されること。
  - プレビューが更新されること。
  - 保存後にメインウィンドウの対象レイヤへ編集済みマスクが反映されること。
  - 保存・破棄・ウィンドウクローズ・深度スライダ変更時の確認が期待どおり動くこと。
